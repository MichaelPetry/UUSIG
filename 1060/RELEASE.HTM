1.1. Level 8R4 - 03 SEP 92

1.1.1 Processor Changes

1.1.1.1 EXEC 43R4 Change

When attempting to run realtime under EXEC 43R4, DBE would get an IGDM error. This has been fixed by correctly loading the realtime level value instead of attempting to access protected memory.

1.1.1.2 AREA Command

Fixed code to clear current page when new area is based.

1.1.1.3 DECODE Command Enhanced / "Q" Processor Option

The DECODE command would error if it encountered an invalid area code in the pointer passed to it. The "Q" processor option was added to allow decode to return a value of zeros instead of erroring.

1.1.1.4 DOUBLE BUFFERED I/O Configuration Change

The size of the buffers used by the double buffered I/O routine (called by SCAN and RSCAN functions) are now configurable and can be specified at DBE SYSGEN time.

1.1.1.5 ERROR HANDLING Fixed

When the error handling routine checked to see if a SCAN was in progress, the register containing the return address was overlayed causing an IGDM error. This has been corrected by saving/restoring the register around the code.

1.1.1.6 FEATURE Command Implemented

Implemented FEATURE command which was documented but never coded.

1.1.1.7 LET Command Fix

Fixed code which caused E$31 errors following SCANs which encountered errors.

1.1.1.8 RSCAN Command Changes

RSCAN now correctly determines whether start/stop pages passed to it are valid.

All arrays passed to the RSCAN function are now monitored.

1.1.1.9 SCAN Command

Changed E$77 error handling to set a bit in ERRSTS table instead.

Added code to detect corrupted page offsets in record slots and to report the condition instead of causing an IGDM error.

Added code to detect record "bit-out" condition, invalid area code or page # in record set pointers.

Cleaned up/standardized error reporting.

1.1.2 System Macro Changes

1.1.2.1 ATT

ATT macro fixed to test queue item status before referencing undefined "qdate" variable.

Calculation of highest track fixed.

Code added to prevent binary find from entering an infinite loop.

1.1.2.2 FRT

FRT macro rewritten to use FRT/DBPDT system tables.

1.1.2.3 LINK

LINK macro changed to recognize an owner record type being encountered and to re-calculate the pointer offset for the set being walked.

1.1.2.4 REC-COUNT

REC-COUNT macro changed to call SCAN$COUNT subroutine macro to display record counts/sizes.

1.1.2.5 SCAN-ONE

SCAN-ONE macro fixed to call SCAN$SIZE macro only once.

Error handling cleaned up. If an error is detected, macro SCAN$ERRMSG will be attempted. This macro can be user written and should reside in the DBE Site Library File.

Average I/O rates were truncated if rates were very large. Macro now uses half words instead of quarter words to hold I/O rate information.

Dropped # of pages on free page chain from I/O calculations.

1.1.2.6 SIZE-REPORT

SIZE-REPORT enhanced to display total tracks used/available for all areas reported on.

1.1.2.7 U$GNLE

Added code to detect zero value in BDI field, indicating BDI of UDS global DBANK is to be used.

Fixed looping problem when TCS GNLE not found.

1.1.3 New System Macros

1.1.3.1 GED

Produces a formatted dump of the Global Encode Table.

1.1.3.2 RFT

Selects the correct FEATURE for the current record and calls the FRT macro.

1.1.3.3 STOC

Call the TOC macro for the current schema file.

1.1.4 Sysgen Changes

1.1.4.1 DOUBLE

DOUBLE BUFFERED IO SIZE IS <tracks> TRACKS

1.6. Level 8R9 - 20 AUG 93

1.6.1 Processor Changes

1.6.1.1 REWRITE Command Enhanced

The "S" security bit, introduced in DBE 8R7, will now allow users to rewrite any schema table but NOT pages or sector formatted disk files.

1.6.1.2 PFFSCH Command

The PFFSCH command allows the user to retreive or update the element table information in the schema file for the currently based schema. The user must have the "S" or "U" security attribute set in order to rewrite the element table entry.

1.6.1.3 PFFSUB Command

The PFFSUB command allows the user to retreive or update the element table information in the subschema file for the currently based subschema. The user must have the "S" or "U" security attribute set in order to rewrite the element table entry.

1.7. Level 9R1 - 25 APR 94

1.7.1 Processor Changes

1.7.1.1 Extraneous Blank Lines From TIP Hardcopy Removed

When sending data via TIP Hardcopy, blank lines would be sent at the end of each buffer write if the last line sent was 33 words long. This has been corrected.

1.7.1.2 Scratch Area Common Bank Added

When DBE initially starts up, a scratch area common bank is aquired. This common bank is used by the SCAN function "Q" option to create the overflow page chain array.

1.7.1.3 KEYIN$ Handler Enhanced

The KEYIN$ interface has been enhanced to provide the current macro line number (if any) and statement currently being executed.

1.7.1.4 HISTOGRAM Negative Number Test

The HISTOGRAM command will now check for negative values. Before this check, DBE would get an IGDM error when encountering a negative value when attempting the HISTOGRAM.

1.7.1.5 MACRO Subtype Supported

A symbolic subtype of MAC is supported by using the "M" option on the OPEN SDFO command.

1.7.1.6 "P" Processor Option Implemented

The "P" processor option has been implemented to control the PAUSE macro when running in demand mode (ON D). If the "P" option is set, the PAUSE macro will function as if an "OFF D" was in effect.

1.7.1.7 RAREA Displays Area Type

When the RAREA command is executed with no parameters, the RDMS area type (DATA,INDEX,DATA/INDEX) will be displayed along with the area name.

1.7.1.8 RSCAN Changes

When RSCAN encounters errors, the word in question will be displayed along with the address on the page.

If the array passed is large enough, the number of records on index pages, in addition to the number of records on data pages, will be returned.

A condition in RDMS 5R1 causes extraneous words of ASCII zeros to be placed at the end of compacted records. The RSCAN function will now take these extra words into account when validating RDMS areas.

1.7.1.9 SCAN Command Enhancements

"I" option added to allow errors detected by SCAN to be ignored. Only a generic "SCAN ERRORS ENCOUNTERED IN AREA..." will be displayed.

"Q" option added to cause the SCAN command to format the overflow pages array in the format of H1=page #,H2=# of overflow pages. This significantly reduces the CPU utilized by the SCAN-ONE macro. In addition, the "Q" option will pass the total IO count back in system variable SCANTOTALIO^.

1.7.1.10 Contigency Error 024512 Trapped

Contingency error 024512 (I/O ERROR WHILE CREATING OUTPUT FILE FOR BREAKPOINTED OR ALTERNATE PRINT/PUNCH FILE) is now trapped with an error message display instead of causing a DBE dump.

1.7.1.11 Error Printing Of Commands With "C" Or "W" Options Fixed

If DBE detected an error on a command or macro call which had the "C" and/or "W" option set, no error would be printed. This has been corrected.

1.7.2 System Macro Changes

1.7.2.1 AREA-PCT

The AREA-PCT will now allow the "S" option to display areas sorted by area name, or the "U"/"D" options to sort up or down by area size.

1.7.2.2 AREAS

PAUSE macro will be called in demand mode after each screen of data is presented.

The "S" option will display areas sorted by area name instead of TIP code.

The "T" option will display whether TIP areas are simplexed or duplexed.

1.7.2.3 BREAK

BREAK will now detect when the end of the input variable is reached instead of scanning past the end.

1.7.2.4 EXPAND$REC

Macro changed to detect when the end of record data is reached.

1.7.2.5 EXPLAIN

Macro now tries to get "explain" text from DBE Help file , DBE Site Library file , User Library file and User Alternate Library file. Use of "On R" reverses this search order.

This macro will search for elements with a version name of "MANUAL" as well as the standard "T$..../EXPLANATION" and "S$...../SYNTAX" elements.

1.7.2.6 MSG-OUT

Use of the "S" option on the macro call will cause the SUPS command to be executed.

1.7.2.7 PAUSE

If the "P" processor option is on, PAUSE will act as if in batch mode.

Specifying "PAUSE ON" or "PAUSE OFF" will set/clear the "P" processor option.

1.7.2.8 PRT

The maximum image size is now truncated to 132 characters to prevent WRITE BUFFER OVERFLOW messages when the "S" option (line number) is used.

1.7.2.9 R$$ANAME

The "T" option will cause the RDMS area type to be printed via a call to the AREA$TYPE macro.

1.7.2.10 R$$FIRSTPAGE

The RDMS area type will be checked to ensure that it is DATA or DATA/INDEX via a call to the R$$VALIDAREA macro.

1.7.2.11 R$$TABLE-LD

Records with compressed keys are now handled.

1.7.2.12 R$BTREE

The RDMS area type is checked to ensure that it is INDEX or DATA/INDEX.

The "Q" option will cause "..." to be printed instaed of data pages.

1.7.2.13 R$COUNT

The RDMS area type is checked to ensure that it is DATA or DATA/INDEX.

1.7.2.14 R$DATA

The computation of record header words has been fixed.

1.7.2.15 R$DUMPREC

Records with compressed keys are now correctly handled.

1.7.2.16 R$FETCH

The start of each data field has been updated with the R$$TABLE-LD macro.

1.7.2.17 R$PGDUMP

Erroneous words of ASCII zeros (from RDMS 5R1) are skipped.

1.7.2.18 R$SCAN

"V" option added to checksum all pages and RAP, RFCP tables.

Both Index and Data record counts are now displayed.

RDMS area type is displayed via the R$$ATYPE macro.

1.7.2.19 R$UPH

This macro has been changed to call the RPH macro with the "U" option.

1.7.2.20 RAPTS

Number of RAP tables is now correctly determined from the File Control Page (RFCP).

1.7.2.21 REC-COUNT

SCAN function "I" option supported.

1.7.2.22 REC-DDL

"W" option added to supress printing of "WITHIN" clauses.

1.7.2.23 RECORDS

Display will PAUSE after each screen of data is presented.

"A" option will cause only records which can be stored in the current area to be reported.

1.7.2.24 RFCP

Enhanced to handle RDMS 5R1 table format.

1.7.2.25 RPH

"U" option added to process "pages" in USRBUF system table.

1.7.2.26 SCAN$FILE

"F" option causes SDF file OUTFL2 to be created containing record counts in SGS format.

1.7.2.27 SCAN$RCOUNT

Record counts which can be stored in area but have no occurrences are now reported.

1.7.2.28 SCAN$TOTREC

Record counts which can be stored in area but have no occurrences are now reported.

1.7.2.29 SCAN-ALL

Record counts which can be stored in area but have no occurrences are now reported.

All options other than "S" are passed to SCAN-ONE.

1.7.2.30 SCAN-ONE

"F" option will be passed to SCAN$FILE macro to create a secondary SDF file containing records counts in SGS format.

"I" option passed to SCAN function (if supplied) to ignore error message printing.

"Q" option will always be used when calling SCAN function. This causes the SCAN function to do most of the overflow page chaining/total IO processing.

All MSG-OUT macro calls changed to include the "S" option.

1.7.2.31 SCHEMA-LOAD

Up to 7 parameters may be passed to SCHEMA-LOAD to be executed.

"A" option added to automatically generate an "AREA" command after all other commands.

"S" option added to automatically generate an "SS" command after all other commands.

1.7.2.32 SCHEMATA

Now calls the SCHEMA-LOAD command to base the schema/area.

1.7.2.33 SETS

"A" option added to report on only those sets which have an owner or member record which can be stored in the current area.

"L" option added to provided long detailed report, including all owner/ member record names.

Display will pause after each screen of data is presented.

1.7.2.34 SYM

"Max lines allowed to be printed" message is ignored if less than max lines are sent.

1.7.2.35 SYNTAX

All processing for the SYNTAX macro is now performed by an internal call to the EXPLAIN macro.

1.7.2.36 SYSTEM

DBE Gen level & date are now displayed.

1.7.2.37 U$APP

DD$FILE , DDL$FILE , DMU$FILE , IRU$FILE , SUDS$FILE system variables are loaded when U$APP is initially called to base an application. The variables are also written to the APPSGS system file in SGS format.

1.7.2.38 U$DAL

Will now correctly handle date/time stamp entries of 0.

1.7.2.39 U$DOWN-LIST

PAUSE added before octal dump if "D" option specified.

1.7.2.40 U$FDT

Calls U$GNLE-DIM to determine table/array sizes.

1.7.2.41 U$FPTE

UDS 5R1 support added.

Calls U$GNLE-DIM to determine table/array sizes.

1.7.2.42 U$GNLE

Calls U$GNLE-DIM to determine table/array sizes.

1.7.2.43 U$USE-COUNT

"C" option causes all messages to be displayed on system console.

"M" option causes messages to be sent every minute waiting for schema to be unused. First message is answerable if "C" option is specified.
